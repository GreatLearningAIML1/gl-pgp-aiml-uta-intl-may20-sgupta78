Context and Defining Problem Statement¶

Background: AllLife Bank wants to focus on its credit card customer base in the next financial year. They have been advised by their marketing research team, that the penetration in the market can be improved. Based on this input, the Marketing team proposes to run personalised campaigns to target new customers as well as upsell to existing customers. Another insight from the market research was that the customers perceive the support services of the back poorly. Based on this, the Operations team wants to upgrade the service delivery model, to ensure that customers queries are resolved faster. Head of Marketing and Head of Delivery both decide to reach out to the Data Science team for help.

Objective: To identify different segments in the existing customer based on their spending patterns as well as past interaction with the bank.

K-Means Clustering: Create appropriate clusters with the scaled data set¶

In order to implement the K-Means clustering, we need to find the optimal number of clusters in which customers will be placed. To find the optimal number of clusters for K-Means, the Elbow method is used based on Within-Cluster-Sum-of-Squares (WCSS) error

In [298]:
cluster_range = range(1,15)
cluster_errors = []

for i in cluster_range:
    clusters = KMeans(n_clusters=i, n_init=5)
    clusters.fit(sdata)
    labels = clusters.labels_
    centroids = clusters.cluster_centers_        # Coordinates of cluster centers
    cluster_errors.append(clusters.inertia_)     # Sum of squared distances of samples to their closest cluster center
    
clusters_df = pd.DataFrame({"num_clusters": cluster_range, "cluster_errors": cluster_errors})

Compare K-Means clusters with Hierarchical clusters¶

K-Means clustering and hierarchical clustering (HC) both share many similarities. Despite that, there are few key differences as well:

1) Techniques for finding optimum number of clusters: K-Means provides Elbow method whereas HC provides dendogram to get the optimum number of clusters.

2) We need to prescribe the number of clusters to get to the stage where we can use Elbow method to get optimum number of clusters however in case of HC, we discover number of optimum clsuters.

3) In case of current dataset HC provided better Silhoutte Score then K-Means.

4) K-Means is based on partition and centroid based approach whereas HC based on hierarchy/tree based approach.

5) K-Means is better option in case of large number of features in the dataset. As you can observe, the plot we used for elbow method, is a simple curve irrespective of number of features whereas dendogram for HC shows so many clusters in the bottom of the dendogram and as number of features increases, it will become even more complicated therefore it will be challanging to visualize the dendogram completely when dataset contains large number of features.
